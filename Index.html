Sistemas de Archivos

Comparación entre Linux (Ext3/4), Windows (NTFS) e iOS (APFS/HFS+)

📚 Reseña Histórica de los Sistemas de Archivos

🐧 Evolución en Linux

1992
Ext (Extended File System) - Primer sistema de archivos nativo para Linux, limitado a 2GB
1993
Ext2 - Mejoró las limitaciones de Ext, soporte hasta 2TB, sin journaling
2001
Ext3 - Añadió journaling para mayor confiabilidad y recuperación de datos
2008
Ext4 - Mejoras en rendimiento, soporte para archivos hasta 16TB y volúmenes de 1EB

🪟 Evolución en Windows

1977
FAT (File Allocation Table) - Sistema básico para disquetes y primeros discos duros
1987
FAT16 - Soporte para discos duros más grandes, hasta 2GB
1996
FAT32 - Eliminó muchas limitaciones de FAT16, soporte hasta 2TB
1993
NTFS (New Technology File System) - Sistema avanzado con seguridad, journaling y compresión

🍎 Evolución en iOS/macOS

1985
HFS (Hierarchical File System) - Primer sistema jerárquico de Apple para Macintosh
1998
HFS+ - Mejoras significativas, soporte Unicode, archivos más grandes
2017
APFS (Apple File System) - Sistema moderno con snapshots, encriptación y optimización para SSD

✅❌ Ventajas y Desventajas

🐧 Linux (Ext3/4)

✅ Ventajas

  • Código abierto y gratuito
  • Altamente estable y confiable
  • Excelente rendimiento
  • Soporte nativo en Linux
  • Journaling desde Ext3
  • Fragmentación mínima

❌ Desventajas

  • Limitado soporte en Windows
  • No tiene compresión nativa
  • Encriptación requiere herramientas externas
  • Ext3 tiene limitaciones de tamaño

🪟 Windows (NTFS)

✅ Ventajas

  • Seguridad avanzada con ACLs
  • Compresión y encriptación nativas
  • Journaling robusto
  • Soporte para archivos grandes
  • Cuotas de disco
  • Recuperación de errores

❌ Desventajas

  • Propietario de Microsoft
  • Fragmentación con el tiempo
  • Soporte limitado en otros OS
  • Overhead de metadatos

🍎 iOS (APFS)

✅ Ventajas

  • Optimizado para SSD/Flash
  • Snapshots instantáneos
  • Encriptación fuerte nativa
  • Copy-on-Write eficiente
  • Clonación rápida de archivos
  • Verificación de integridad

❌ Desventajas

  • Exclusivo de Apple
  • Relativamente nuevo
  • Sin soporte en otros sistemas
  • Limitado a dispositivos Apple

💻 Comandos Principales por Sistema

🐧 Comandos Linux

Crear directorios:
mkdir directorio
mkdir -p ruta/completa/directorio
mkdir -m 755 directorio_con_permisos
Cambiar directorios:
cd directorio
cd ..
cd ~
cd -
pwd (mostrar directorio actual)
Asignar permisos:
chmod 755 archivo
chmod u+x archivo
chmod -R 644 directorio
chown usuario:grupo archivo
chgrp grupo archivo
Copiar archivos:
cp archivo destino
cp -r directorio destino
cp -p archivo destino (preservar atributos)
rsync -av origen destino
Mover archivos:
mv archivo destino
mv archivo nuevo_nombre
mv directorio nuevo_directorio
Otros comandos útiles:
ls -la (listar archivos)
rm archivo (eliminar)
rm -rf directorio (eliminar directorio)
find / -name archivo
du -h directorio (tamaño)
df -h (espacio en disco)

🪟 Comandos Windows (CMD/PowerShell)

Crear directorios:
mkdir directorio
md directorio
New-Item -ItemType Directory -Path "ruta" (PowerShell)
Cambiar directorios:
cd directorio
cd ..
cd \
cd %USERPROFILE%
pwd (PowerShell)
Asignar permisos:
icacls archivo /grant usuario:F
icacls archivo /deny usuario:R
takeown /f archivo
attrib +r archivo (solo lectura)
Copiar archivos:
copy archivo destino
xcopy /s /e origen destino
robocopy origen destino /E
Copy-Item archivo destino (PowerShell)
Mover archivos:
move archivo destino
ren archivo nuevo_nombre
Move-Item archivo destino (PowerShell)
Otros comandos útiles:
dir (listar archivos)
del archivo (eliminar)
rmdir /s directorio
where archivo (buscar)
fsutil volume diskfree C: (espacio)

🍎 Comandos iOS/macOS (Terminal)

Crear directorios:

⚡ Comparación de Rendimiento

Factores de Rendimiento

  • Ext4: Excelente para operaciones secuenciales, baja fragmentación, journaling eficiente
  • NTFS: Bueno para archivos grandes, puede fragmentarse, compresión transparente afecta rendimiento
  • APFS: Optimizado para SSD, copy-on-write puede generar overhead, snapshots muy eficientes

Casos de Uso Recomendados

  • Ext4: Servidores Linux, estaciones de trabajo, sistemas embebidos
  • NTFS: Escritorios Windows, servidores Windows, almacenamiento empresarial
  • APFS: Dispositivos Apple modernos, almacenamiento SSD, backup con Time Machine
